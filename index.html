实验目的：利用主流云平台搭建个人博客或网站

实验目的

熟悉使用主流云平台并搭建个人博客或者网站。 参考方案：基于主流云平台，设计、实现个人博客或者网站的搭建，撰写实验报告（附带网站链接并可以访问），并在网站上呈现此次实验报告。

硬件环境

联网的计算机一台

软件环境

Windows or Linux

实验内容与步骤

注册并登录GitHub账号：如果你还没有GitHub账号，需要先注册一个账号并登录。

创建一个新的仓库：在GitHub首页，点击右上角的加号图标，然后选择"New repository"（新仓库）。

设置仓库信息：在弹出的窗口中，填写仓库的名称、描述等信息

添加README文件：在创建仓库时，选择“Add a README file”（添加一个README文件）。这将有助于他人更好地了解你的博客内容。在README文件中，可以添加博客的主题、结构、文章列表等信息。

创建博客主题：使用现有的博客主题或自己编写一个。如果使用现有主题，在仓库的根目录下创建一个名为“Hexo-blog”的文件夹，并将主题文件放入其中。如果自行编写主题，也需要将源代码放入“Hexo-blog”文件夹。

配置博客：在仓库的根目录下，创建一个名为“config”的文件夹。在里面添加一个名为“config.yml”的文件，用于配置博客。你可以参考现有的配置文件，设置博客的基本信息，如标题、描述、分页数量等。

克隆博客到本地：在计算机上安装Git，然后使用以下命令将博客克隆到本地： git clone https://github.com/your-username/my-blog.git

结论分析与体会

知道了GitHub博客基于静态站点生成器，如Jekyll、Hexo等，这意味着无需安装和配置复杂的运行环境。博客的部署和更新可以通过简单的命令行或脚本自动化完成，降低了维护成本。

学会了GitHub博客可以使用众多优秀的开源主题和插件，如Hexo的插件生态非常丰富，涵盖了搜索、评论、分类、归档等功能。这使得GitHub博客具有较高的可定制性和扩展性

了解到了Hexo是一个快速、简单且强大的博客框架，适用于建立个人或团队博客。在创建个人网站时，可以考虑使用Hexo，因为它具有易于安装、快速搭建、丰富的主题和插件等优点。
